# DESIGN.md — 7일 7솔 디자인 원칙 (Apple 지향)

이 프로젝트의 UI/UX는 Apple의 **Human Interface Guidelines** 철학을 따른다. 화려함이 아니라
**명료함**으로 사용자를 돕는다. 아래는 우리가 코드로 지킬 구체적 기준이다.

## 1. 세 가지 기둥 (Apple의 핵심 원칙)

- **Clarity(명료성)** — 텍스트는 어떤 크기에서도 읽히고, 아이콘은 정확하며, 장식은 절제한다.
  기능이 콘텐츠를 통해 드러나게 한다. "이게 뭐지?"가 없어야 한다.
- **Deference(양보)** — UI는 콘텐츠에 자리를 내준다. 크롬(테두리·그림자·버튼)은 최소화하고,
  여백과 타이포그래피로 위계를 만든다. 앱이 아니라 **사용자의 데이터**가 주인공.
- **Depth(깊이)** — 레이어와 부드러운 모션으로 계층과 맥락을 전달한다. 전환은 의미가 있어야 하고,
  절대 산만하지 않다.

## 2. 타이포그래피

- **폰트**: 시스템 폰트 우선 — `-apple-system, BlinkMacSystemFont, "SF Pro Text", system-ui,
  "Apple SD Gothic Neo"`. (한글은 Apple SD Gothic Neo)
- **위계는 무게와 크기로**, 색으로 억지로 만들지 않는다.
  - Display/Hero: 40–56px, `font-weight: 600`, `letter-spacing: -0.02em`
  - Title: 20–28px, 600
  - Body: 15–17px, 400, `line-height: 1.47`
  - Caption/Secondary: 12–13px, `--text-secondary`
- 제목은 자간을 살짝 좁혀(-0.02em) Apple 특유의 조밀함을 낸다.
- 문장부호로 소리치지 않는다. 느낌표·전부대문자 지양.

## 3. 색

절제된 팔레트. 회색조 위에 **단 하나의 강조색(systemBlue)**.

| 토큰 | Light | Dark |
|------|-------|------|
| `--bg` (페이지) | `#f5f5f7` | `#000000` |
| `--surface` (카드) | `#ffffff` | `#1c1c1e` |
| `--text` | `#1d1d1f` | `#f5f5f7` |
| `--text-secondary` | `#6e6e73` | `#86868b` |
| `--border` | `rgba(0,0,0,.08)` | `rgba(255,255,255,.12)` |
| `--accent` (systemBlue) | `#0071e3` | `#0a84ff` |
| `--success` | `#34c759` | `#30d158` |
| `--warning` | `#ff9500` | `#ff9f0a` |
| `--danger` | `#ff3b30` | `#ff453a` |

- 강조색은 **행동(버튼·링크·진행)** 에만. 배경을 강조색으로 칠하지 않는다.
- **라이트/다크 모두 지원** — 시스템 설정(`prefers-color-scheme`)을 존중한다. `color-scheme: light dark`.

## 4. 형태 · 공간

- **넉넉한 여백이 곧 디자인.** 요소를 채우지 말고 숨 쉬게 둔다. 섹션 간 40–64px, 카드 내부 20–28px.
- **부드러운 모서리**: 카드 18px, 인풋/작은 요소 12px, 버튼은 pill(980px)에 가깝게.
- **그림자는 아주 은은하게** — 떠 있는 느낌만. `0 1px 3px rgba(0,0,0,.06), 0 8px 24px rgba(0,0,0,.06)`.
  다크모드에선 그림자 대신 표면 밝기 차로 깊이를 낸다.
- 테두리는 1px, 저대비. 선으로 가두지 말고 여백으로 나눈다.
- 콘텐츠 폭은 읽기 좋은 상한(본문 640–720px)으로 제한하고 가운데 정렬.

## 5. 컨트롤 · 인터랙션

- **버튼**: primary는 systemBlue 채움 + 흰 글씨, 라운드. secondary는 무채색 외곽선/텍스트.
  터치 타깃 최소 44px 높이 감각.
- **인풋**: 표면색 배경 + 저대비 테두리, 포커스 시 강조색 링(2px). 플레이스홀더는 secondary.
- **피드백은 즉각적이고 조용하게** — 로딩은 스피너 대신 은은한 상태 텍스트/스켈레톤,
  성공은 체크 하나로.
- **모션**: 120–260ms, `cubic-bezier(0.4, 0, 0.2, 1)` 같은 ease-out. 등장은 살짝 위로+페이드.
  과한 바운스·회전 금지.

## 6. 웹 vs 확장 — 같은 언어, 다른 방언

원칙은 공유하되 **역할에 맞게 다르게** 적용한다. (둘을 한 화면처럼 만들지 않는다.)

| | 웹 (스터디 허브) | 확장 (인-컨텍스트 도구) |
|--|------------------|--------------------------|
| 성격 | 펼쳐 보고 관리 | 순간 액션, 흐름 방해 최소 |
| 밀도 | 여유·큰 타이포·카드 | 초경량, 요소 최소 |
| 표면 | Apple식 정돈된 카드 레이아웃 | LeetCode 위 오버레이(그 페이지에 얹힘) |
| 색 | 무채색 + systemBlue 강조 | 상태색 강하게(초록=Accepted 등) — **눈에 띄어야** 제 역할 |
| 모션 | 은은한 페이드/슬라이드 | 딱 필요한 피드백(성공 펄스) 한 번 |
| 목표 | 아름답고 조용함 | 빠르고 확실함 |

웹은 "머무는 곳"이라 Apple의 차분한 미학을 최대로. 확장은 "찌르는 도구"라 미니멀하되
상태 전달은 과감하게. 이 차이를 의도적으로 유지한다.

## 7. 접근성

- 본문 대비 4.5:1 이상, 큰 텍스트 3:1 이상.
- 색만으로 정보 전달 금지 — 텍스트/아이콘 병기(예: 달성 "✓ 달성").
- 포커스 링 항상 보이게. 키보드로 모든 동작 가능.
- 모든 인터랙티브 요소에 적절한 라벨.

---

> 한 줄 요약: **"콘텐츠가 주인공, UI는 조용한 조력자."** 회색조 위 하나의 파랑,
> 넉넉한 여백, 시스템 폰트, 은은한 깊이. 웹은 차분하게 머물게, 확장은 빠르게 찌르게.
